# Развертывание ВМ на локальной машине
Для автоматичиского создания виртуальной машины, нужно воспользоваться написанным bash-скриптом `alma_auto.sh`, который создает ВМ в VirtualBox. При этом предварительно нужно скачать ISO-образ AlmaLinux и пакет `xorriso` для манипуляций с самим образом.
Перед началом работы скрипта, можно и даже нужно настроить его под себя, поменяв нужные переменные:
1. `name` (имя ВМ);
2. `cores` (кол-во ядер)
3. `memory` (ОЗУ в мбайт)
4. `size` (память виртуального диска в мбайт)
5. `bridge` (сетевой адаптер)
6. `vm_folder` (папка создаваемой ВМ)
7. `iso` (путь до ISO-образа, первый аргумент скрипта)

Создание ВМ скриптом:
```
$ ./alma_auto </path/to/iso>
```

# Развертывание микросервиса в облаке с Terraform
Предварительно настроим terraform. В домашней директории создадим файл `.terraformrc`, где укажем зеркало для yandex провайдера:
```
provider_installation {
    network_mirror {
        url = "https://terraform-mirror.yandexcloud.net/"
        include = ["registry.terraform.io/*/*"]
    }
    direct {
        exclude = ["registry.terraform.io/*/*"]
    }
}
```
Предварительно нужно создать учетную запись в yandex cloud и желательно скачать утилиту yandex cloud cli.

После установки пройти регистрацию в командной строке yandex cloud cli:
```
$ yc init
```
Cгенерировать временный токен для работы:
```
$ yc iam create-token
```
Инициализировать терраформ (подгрузить провайдеры и модули):
```
$ terraform init
```
Полученный `cloud-id`, `folder-id,` `token` нужно вставить в файл `credentials.tf`. Также лучше переменные `key_dir`, `key_name` настроить под себя. `key_dir` каталог в котором будет храниться пара ключей, `key_name` имя ключей.

Перед подтверждением развертывания лучше проверить конфигурацию с помощью команды:
```
$ terraform plan
```
После можно применить конфигурацию:
```
$ terraform apply
```

***Примечание***: если используется Terraform, то в каталоге ansible автоматически создается инвентарь inventory.ini с созданной ВМ.

# Развертывание микросервиса на ВМ
Развертывание сервиса реализуется с помощью Ansible плейбуков. Если мы развертываем на созданном вм, без использования облаков, то нужно открыть 8080/tcp порт.
```
$ sudo firewall-cmd --zone=public --permanent --add-port=8080/tcp
```
Так же можно воспользоваться написанным плейбуком `vm-settings.yml`:
```
$ ansible-playbook vm-settings.yml
```

Cоздадим Ansible инвентарь, укажем к каким устройствам хотим обращаться (при развертывании с помощью terraform инвентарь создается автоматически):

***Пример инвентаря***:
```
<host> ansible_host=<ip> ansible_user=<username> ansible_ssh_private_file=<path/to/private/key>
```

Дальше нужно скопировать сам микросервис на ВМ, а также настроить его. Используем плейбук `service_exporter.yml`. Перед применением нужно поменять пути до файлов под сети (переменные `source`, `destination`, `config_source`)
```
$ ansible-playbook service_exporter.yml
```

# Развертывание сервиса в Контейнере
Предварительно необходимо установить нужные инструмены docker. Для этого можно воспользоваться плейбуком `docker_alma.yml`.
```
$ ansible-playbook docker-alma.yml
```

Развертывание контейнера реализуется с помощью плейбука `docker_exporter.yml`. Предварительно нужно поменять переменные `source`, `destination`, в которых нужно указать путь до каталога docker со всеми зависимостями и место, куда будет скопирован каталог source соответственно:
```
$ ansible-playbook docker_exporter.yml
```
